# Alarm Datetime Menu Chunk Regression

## Symptom

After deploying `river-watch-alarm-list-full-datetime-increment-20260714.zip`, the Alarm Center loaded and showed the new full date, but left-navigation items did not open their pages.

## Root Cause

The v1 package replaced `index.html` and the main bundle with `index-BLFEVMej.js`. That main bundle references 20 route chunks generated by the same Vite build. The compact package shipped only `AlarmsPage-PDfGuYHo.js`; 36 referenced JavaScript and CSS assets were absent from the package. The server retained chunks from a different build, so route clicks requested non-existent hashed files.

## Reproduction

```powershell
node codex-handoff/scripts/verify-frontend-dist-closure.mjs `
  river-watch-alarm-list-full-datetime-increment-20260714/frontend/dist
```

Expected result for v1: non-zero exit with missing page chunks.

## Correction

`river-watch-alarm-list-full-datetime-increment-20260714-v2.zip` ships the complete `frontend/dist/assets` directory from the verified build, while continuing to exclude large unrelated `digital-twin` resources. The deployment script replaces the assets directory as one versioned set and verifies every deployed file hash.

## Prevention

Every future frontend package that replaces `index.html` or the main Vite entry must pass `verify-frontend-dist-closure.mjs`. A main entry and its hashed chunks are an atomic release unit; they must not be selected independently by apparent feature ownership.
